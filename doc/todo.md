初期测试：预期直接使用qrcode模块尝试生成，问题在于缺少起始终止标记，<br>或许可以考虑前后帧为全白，由白进入有用信息时判定为开始，再次进入白时<br>，判定为结束

```markdown
[标识符][块编号/总块数][CRC][数据]

标识符：1 字节
块编号 + 总块数：各 2 字节 → 共 4 字节
CRC32：4 字节
合计：1 + 4 + 4 = 9 字节

- 标识符：可以是 START / DATA / END。包含起始，数据，结束，冗余
- 块编号：例如 当前块号/总块数，这样即使丢掉起始块，也能知道总量。
- CRC：块级别校验。
- 数据：真实负载。
```
